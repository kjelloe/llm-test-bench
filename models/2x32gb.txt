# Dual 32 GB VRAM model set — 2× RTX 6000 Ada / A6000 / RTX 5090 etc. (64 GB total)
# All models use tensor_split=1|1 (equal layer distribution across both GPUs).
#
# Purpose: 70B+ models that cannot fit on a single GPU or on 2×24 GB.
# 30B/35B models (qwen3-coder, qwen3.5:35b, noctrex) are NOT listed here — they already
# receive their full KV benefit at 48 GB (2x24gb.txt); repeating them at 64 GB adds no insight.
#
# Format: <ollama-name>  [<gguf-file>  [<params>]]  [hf:<owner/repo>]
#   gguf-file  GGUF filename relative to $LLAMA_MODELS_DIR (for --backend llama-server)
#   params     comma-separated llama-server flags
#   hf:        HuggingFace repo for fetch-hf.sh download helper
#
# IMPORTANT: speeds below are estimates — not yet benchmarked on dual-GPU hardware.
# NVLink vs PCIe between GPUs significantly affects 70B tensor-parallel throughput.

# Speed index (est.):
# qwen2.5:72b    #  ~20 tok/s  dense 72B Q5_K_M; ~48 GB weights, ~8 GB/GPU for KV
# llama3.3:70b   #  ~20 tok/s  dense 70B Q5_K_M; ~46 GB weights, ~9 GB/GPU for KV
# gpt-oss:120b   #  stress test — mxfp4 ~60 GB; fully GPU-resident on 64 GB (hybrid RAM+GPU on 24 GB)
