# 8 GB VRAM model set — RTX 4060 Ti 8GB, RTX 3060 12GB (8 GB effective), etc.
# Target: models whose weights fit in ≤6 GB, leaving ≥2 GB for KV cache.
#
# Format: <ollama-name>  [<gguf-file>  [<params>]]  [hf:<owner/repo>]
#   gguf-file  GGUF filename relative to $LLAMA_MODELS_DIR (for --backend llama-server)
#   params     comma-separated llama-server flags
#   hf:        HuggingFace repo for fetch-hf.sh download helper
#
# Context ceiling: 8 GB cards are very VRAM-constrained.
# - 6 GB model (e.g. mellum2) leaves ~2 GB for KV → max ~16–32k tokens at q8_0 KV
# - 4.5 GB model (e.g. 7B Q4) leaves ~3.5 GB for KV → max ~32–48k tokens at q8_0 KV
# context_64k+ will CTX_TRUNCATED on real 8 GB hardware for all models in this file.
# The max_ctx values here are set for 8 GB headroom.
