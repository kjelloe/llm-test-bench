# ── Nemotron-3-Nano 30B-A3B ──────────────────────────────────────────────────
# Hybrid MoE: 23 Mamba-2/MoE layers + 6 Attention layers. 3.5B active / 30B total.
# Q4_K_M ~18 GB — fully GPU-resident on 24 GB with ~6 GB headroom.
# Thinking-capable via chat template flag (same pattern as Qwen3).
# KV cache overhead ≈ 1/5 of a pure transformer (only 6 attention layers accumulate KV):
#   max_ctx=131072 is safe; 256k or higher likely feasible — needs testing.
# Benchmarks (NVIDIA, vs qwen3-coder:30b proxy Qwen3-30B-A3B-Thinking-2507):
#   AIME25 89.1 (vs 85.0), LiveCodeBench 68.3 (vs 66.0), SWE-Bench 38.8 (vs 22.0)
# Results (2026-05-15, llama-server, RTX 3090 24 GB, coding tasks):
#   12/15  168.3 tok/s  ~274s total  (confirmed — rerun of 3 fails at 144–151 tok/s all TESTS_STILL_FAIL)
#   PASS: node_slugify, python_safe_div, dotnet_sas, node_csv_parser, python_lru_cache,
#         python_lfu_cache, python_minheap, python_multifile_rename, node_memoize_bug,
#         python_ledger_bug, python_dijkstra, python_hashmap (L5 — unique pass among thinking models)
#   FAIL: csv_nordic_property (incorrect logic, not budget — 8.9k tokens generated, still wrong)
#         python_expr_eval, python_tokenizer (L4 — genuine capability gap; 3k/5k tokens, wrong output)
# Fastest thinking model tested: 168 tok/s beats qwen3.5:35b (133), qwen3.6:35b-A3B (141), qwen3-coder:30b (152).
# Context results (2026-05-15): context_8k PASS 166.7 tok/s, context_32k PASS 144.9 tok/s,
#   context_64k PASS 116.6 tok/s, context_128k CRASH (server OOM at ctx=131072).
# Mamba-2 KV advantage is real: 116 tok/s at 64k vs qwen3-coder:30b at 3.8 tok/s at 128k.
# Safe limit on 24 GB is ctx=65536. Tested q8_0 and q4_0 KV — both OOM at ctx=131072.
# KV cache precision is not the bottleneck: llama.cpp appears to pre-allocate Mamba-2
# activation/state buffers proportional to context length (not just the 6 attention layers).
# This is a llama.cpp bug — should be filed upstream. max_ctx=65536 is the confirmed safe limit.
