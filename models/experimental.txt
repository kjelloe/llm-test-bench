# Three non-MoE weight variants in the same GGUF repo:
#   BF16 — 21.39 GB  (best accuracy; use if GPU supports native BF16)
#   F16  — 21.39 GB  (faster on older architectures that emulate BF16)
#   Q8   — 19.71 GB  (recommended for 24 GB — leaves ~4 GB for KV cache)

#   --spec-type draft-mtp        enable self-speculative decoding
#   --spec-draft-p-min 0.75      only speculate when draft prob >= 0.75 (new vs carnice)
#   --spec-draft-n-max 3         draft 3 tokens per step
#
# ⚠ DETERMINISM WARNING (from carnice:35b experience):
#   spec-type draft-mtp broke temp=0 determinism for carnice — python_hashmap REGRESSED.
#   --spec-draft-p-min 0.75 is intended to reduce acceptance noise but does not
#   guarantee greedy-equivalent output. Benchmark WITHOUT spec flags first; add them
#   only for throughput comparison runs where result correctness is not the primary goal.

#   (llama-server GGUF params use underscores; exact flag name may differ — verify with
#    llama-server --help if the spec flags are not recognised)
#
# Compared to carnice:35b: same MTP architecture, different base quant (MXFP4 vs Q4_K_M).
